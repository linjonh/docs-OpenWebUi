在 Open WebUI，我们致力于不断增强我们的平台，为用户提供最佳体验。以下是我们正在进行和未来开发的结构化路线图，分为界面、信息检索和社区三大类别。

## 界面 🖥️

我们的界面创新路线图旨在打造一个高度直观且易于使用的平台。这不仅提升用户满意度，还通过减少用户认知负担提高生产力。

- 📦 **单一二进制可执行文件**：简化部署并确保在不同平台上的兼容性，我们计划将界面提供为单一二进制可执行文件。这种方式将使安装和更新变得轻松，对特别是技术知识有限的用户显著改善可用性。

- 👤 **用户页面**：一个个人用户页面功能，允许用户创建帖子，并包含关注者、点赞与评论功能。这让用户能够有效地与更广泛的社区分享自己的模型配置、提示和文件，从而在平台周围形成一个更丰富、更有连接性的生态系统。

- 📝 **AI驱动的笔记**：受工具如 Notion 和 Obsidian 的启发，我们计划推出一个强大的笔记功能，包括AI集成。从简单的笔记到正式的文档创作，这个工具将在平台内本地实现无缝体验。

- 📈 **高级用户追踪和成本管理工具**：用户可获得为应用性能和用户活动追踪设计的综合工具，以及有效管理成本的工具。这些工具将为用户提供所需的数据来做出明智的决策，改善用户体验，并保持预算控制，从而优化其AI应用资源的使用效果。

- 🧠 **AI工作流工具**：一个基于节点的工具，用于编排并组合AI系统的多个方面。该工具允许用户直观地连接不同的AI模块和服务，轻松创建复杂的工作流。它旨在使用户能够充分利用AI的潜力，而不需要掌握深奥的AI编程技术。

- 🔊 **本地文本转语音集成**：此功能将允许自然语言处理将文本转换为逼真的语音音频，从而使平台更具可访问性，特别是对视觉障碍用户。此集成有助于无需屏幕交互消费信息，促进多任务处理并提高用户参与度。

- 📣 **唤醒词检测**：通过语音命令实现免操作模式，不仅增强了可访问性，还符合物联网和智能环境的最新趋势，使我们的平台为未来做好准备。

- 💻 **更佳的代码执行**：增强平台内的编码功能，可支持多种开发活动，使其成为开发人员的一站式解决方案，吸引更广泛的技术受众并促进强大的开发者社区。

- 🤖 **代码解释器功能**：嵌入到平台内部的灵活代码解释器可支持多种编程语言，从而减少对外部工具的依赖并简化开发流程，提高效率。

- 🔧 **精简微调支持**：Open WebUI 将通过界面对评价响应自动生成微调数据集，为用户提供一个简化的微调流程。我们将提供 Python 笔记本来帮助预处理您的数据，使其通过笔记本直接完成微调。这种方式将非常简单，并优雅地将微调整合到用户的工作流中。


## 信息检索 (RAG) 📚

Open WebUI 当前提供了一个界面，此界面能够高效使用现有的信息检索(IR)模型。然而，我们认为仍然有巨大的改进空间。我们完全认识到当前WebUI中RAG（基于检索增强生成）的实现的局限性，理解到一刀切的方法无法有效满足所有用户的需求。

为了改善这一点，我们承诺将框架转变为高度模块化并且广泛可配置。这一新的方向将使用户能够通过简单直观的界面轻松地定制信息检索过程，借助拖放即可完成设置。这一优化旨在显著改善用户体验，通过提供更加灵活且个性化的信息检索设置。

以下是我们专注于推进信息检索能力的计划：

- 🌐 **可定制的RAG框架**：展望未来，您将能够通过一个用户友好的界面轻松定制您的信息检索设置。这将包括模块化组件，用户无需深厚的技术知识即可拖放并配置。

- 🔄 **与最先进方法的高级集成**：我们的目标是整合最新的信息检索方法，通过先进技术和架构提升准确度和效率。

- 🔍 **专注的研发**：我们正在加大研发力度，以发现并整合能够突破现有方法界限的新型检索方法，与领先的研究机构合作。

## 社区 🤝

我们旨在创建一个充满活力的社区，每个人都可以贡献并从共享知识和开发中受益：

- 🏆 **LLM排行榜**：一种社区驱动的评估和排名模型的方法，通过实际应用来对性能进行基准测试，而不是依赖于可能不反映实际效用的标准数据集，以确保透明度和持续改进。

- 👥 **子社区**：鼓励细分社区策划数据集和优化提示，这样可以获得更好调优的模型，因为这些社区拥有独特的见解和详细的反馈，这通常在广泛的数据集中被忽略。

🔗 **社区白皮书**：探索我们的详细策略，并在我们的 [白皮书](https://openwebui.com/assets/files/whitepaper.pdf) 中与我们一起规划AI驱动平台的未来。
