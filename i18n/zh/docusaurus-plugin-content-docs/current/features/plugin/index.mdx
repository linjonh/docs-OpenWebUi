# 🛠️ 工具与功能

想象一下，你刚刚发现了 Open WebUI，或者你已经在使用它了，但对于关于“工具”，“功能”和“管道”的各种说法感到有些迷茫。这一切听起来像是某种神秘的技术术语，对吧？别担心！让我们一步一步，超级清晰地详细解析这些概念。结束后，你会对这些术语的含义、它们的工作原理以及为什么它们其实并不那么复杂有一个清晰的了解。

## 简而言之

- **工具**扩展了 LLM 的能力，使其能够收集现实世界的实时数据，比如天气、股价等。
- **功能**扩展了 Open WebUI 的本身功能，使你能够添加新的 AI 模型支持（比如 Anthropic 或 Vertex AI）或者提升可用性（比如创建自定义按钮或过滤器）。
- **管道**更适用于高级用户，他们希望将 Open WebUI 的功能转变为 API 兼容的工作流——主要用于卸载繁重的处理任务。

入门工具和功能很简单，因为一切都已经内建在核心系统中！你只需要**点击一个按钮**并**直接从社区导入这些功能**，无需编写代码或深入的技术操作。

## 什么是“工具”和“功能”？

我们先把 **Open WebUI** 想象成一个可以处理与大语言模型（LLM）相关的许多任务的“基础”软件。但有时候，你需要一些额外的功能或能力，这些并非“开箱即用”——这就是 **工具** 和 **功能** 的用武之地。

### 工具

**工具**是一个令人兴奋的功能，因为它们允许 LLM 做的不仅仅是处理文本。它们提供了 **外部能力**，这是 LLM 自身无法拥有的。

#### 工具示例：

想象一下，你正在与一个 LLM 交流，你希望它提供最新的天气更新或实时股价。通常，LLM 是无法做到这一点的，因为它只是基于预训练的知识工作。这时候 **工具** 就派上用场了！

- **工具就像插件**，LLM 可以使用它们来收集 **现实世界的实时数据**。因此，启用了“天气工具”后，模型可以上网收集实时天气数据，并在你的对话中显示出来。

工具本质上是你赋予 AI 的一种 **能力**，帮助它与外界互动。通过添加这些工具，LLM 可以“抓取”有用的信息或根据对话的上下文执行专门的任务。

#### 工具示例（扩展 LLM 的能力）：
1. **实时天气预测** 🛰️。
2. **股价检索** 📈。
3. **航班跟踪信息** ✈️。

### 功能

虽然 **工具** 是 AI 在对话中使用的，**功能** 则可以帮助扩展或自定义 Open WebUI 本身的功能。想象一下，工具像是在菜肴中添加新食材，功能则是控制厨房的过程！🚪

#### 简单解释：

- **功能**让你能够调整或添加 **Open WebUI 内部的功能**。
- 你不是给 LLM 增加新的能力，而是扩展平台本身的 **界面、行为或逻辑**！

例如，你可能希望：
1. 在 WebUI 中添加一个新的 AI 模型，比如 **Anthropic**。
2. 在工具栏中创建一个自定义按钮，用来执行经常使用的命令。
3. 实现一个更好的 **过滤功能**，捕捉来自输入文本的不适当或**垃圾信息**。

没有功能，这些都无法实现。但通过 Open WebUI 中的这个框架，你可以轻松扩展这些功能！

### 差异总结：
- **工具** 是允许 LLM 在其默认能力之外**做更多事情**（例如检索实时信息或根据外部数据执行自定义任务）。
- **功能** 是帮助 WebUI 本身**做更多事情**，比如添加新的 AI 模型或创建更智能的过滤数据方法。

两者设计为 **可插拔**，这意味着你可以通过社区轻松导入它们，只需单击一次！🎉 你无需花费数小时编写代码或进行复杂操作。

## 什么是管道？

然后，我们介绍 **管道**……这部分听起来开始有些技术性——但别担心。

**管道** 是 Open WebUI 的一项计划，旨在使 WebUI 的每个部分都能与 OpenAI 的 API 系统 **互操作**。本质上，管道扩展了 **工具** 和 **功能** 已经做到的事情，但现在提供更多灵活性。它们允许你将功能转换为兼容 OpenAI API 的格式。🧠

### 但问题是……

你可能 **不需要** 管道，除非你正在处理超级高级的设置。

- **管道适合谁？** 通常是适合 **专家** 或运行更复杂用例的人。
- **它们什么时候需要用到？** 如果您希望将处理工作从主要的 Open WebUI 实例转移到另一台机器（以防止过载您的主要系统），那么它们就派上用场了。
  
在大多数情况下，无论是初学者还是中级用户，您都不用担心管道问题。只需专注于享受 **工具** 和 **功能** 给您的 Open WebUI 体验带来的收益即可！
