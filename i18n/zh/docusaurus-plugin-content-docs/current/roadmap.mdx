在Open WebUI，我们致力于不断增强我们的平台，为用户提供最佳体验。以下是我们未来以及正在进行的开发计划的结构化路线图，分为界面、信息检索和社区三个分类。

## 界面 🖥️

我们的界面创新路线图旨在打造一个高度直观和易用的平台。这不仅能够提升用户满意度，还能通过降低用户的认知负担来提高生产力。

- 📦 **单一二进制可执行文件打包**：简化部署并确保跨不同平台的兼容性，我们计划将界面以单一二进制可执行文件形式提供。这种方法将使安装和更新变得简单，显著提升用户的易用性，尤其是技术背景有限的用户。

- 👤 **用户页面**：一个个人用户页面功能，允许用户创建帖子。这项功能还将包含关注者、点赞和评论等功能。这能让用户有效地与更广泛的社区分享他们的模型配置、提示和文件，从而创造一个更丰富、更紧密连接的平台生态系统。

- 📝 **AI驱动的笔记功能**：受工具如Notion和Obsidian的启发，我们计划引入一个包含AI集成的强大笔记功能。从简单记笔记到完整的文档创建，这个工具将提供一个无缝体验，并完全在平台上本地集成。

- 📈 **高级用户跟踪和成本管理工具**：用户将能够使用设计精美的工具来跟踪应用性能和用户活动，以及有效管理成本。这些工具将赋予用户所需的数据，以便做出明智的决策、改善用户体验，并控制预算，从而优化AI应用中的资源使用。

- 🧠 **AI工作流工具**：一个基于节点的工具，用于编排和组合多个AI系统的方面。这个工具将使用户能够可视化地连接不同的AI模块和服务，从容创建复杂工作流。它旨在赋能用户，让他们无需深刻的AI编程技术知识，也能够充分利用AI。

- 🔊 **本地文本到语音集成**：这项功能将允许自然语言处理将文本转化为逼真的语音音频，从而使平台更加易用，尤其是对于视觉障碍的用户。这种集成帮助用户无需屏幕互动即可消费信息，方便多任务处理并提高用户参与度。

- 📣 **唤醒词检测**：通过语音命令实现无手操作，这不仅增强了可访问性，还符合物联网和智能环境的最新趋势，使我们的平台具有未来适应性。

- 💻 **更好的代码执行功能**：提升平台的编码功能，以支持广泛的开发活动，打造开发者的一站式解决方案，从而吸引更多技术观众并培养一个健壮的开发者社区。

- 🤖 **代码解释器功能**：一个灵活的代码解释器嵌入到平台中，可支持多种编程语言，这降低了对外部工具的依赖，简化了开发流程并提高效率。

- 🔧 **流线型微调支持**：Open WebUI将提供一个无缝微调过程，允许用户仅通过界面和评分回复来自动创建微调数据集。我们将提供Python笔记本来帮助预处理您的数据集，使之可以直接从笔记本进行微调。这种方法超级简单，并优雅地将微调集成到用户的工作流中。


## 信息检索（RAG） 📚

Open WebUI当前提供了一个界面，能够高效地使用现有的信息检索（IR）模型。然而，我们认为仍然有很大的改进潜力。我们完全意识到我们当前在WebUI中的RAG（检索增强生成）实现的局限性，理解一种适用于所有的方式无法有效满足所有用户需求。

为此，我们承诺将我们的框架转变为高度模块化并且广泛可配置。这个新方向将使用户能够轻松地根据自己的特定需求定制信息检索过程，通过一个简单、交互式的界面来实现拖放式的自定义。这种优化旨在通过提供更加适应和个性化的IR设置显著提升用户体验。

以下是我们提升信息检索能力的重点计划：

- 🌐 **可自定义的RAG框架**：设想一个未来，您可以通过用户友好的界面轻松定制您的信息检索设置。这将包括模块化组件，用户可以拖放和配置，而无需深入的技术知识。

- 🔄 **与最新方法的高级整合**：我们致力于引入检索方法的最新进展，通过先进技术和架构提升准确性和效率。

- 🔍 **专注于研发**：我们将加大研发力度，发现并整合突破现有方法的新型检索方式，并与领先的研究机构合作。

## 社区 🤝

我们计划打造一个活跃的社区，每个人都能贡献或从共享的知识与成果中受益：

- 🏆 **LLM排行榜**：社区驱动的模型评估和排名方式能保证透明性和持续改进，利用真实世界应用来基准测试性能，而不是依赖可能无法反映实际实用性标准的数据集。

- 👥 **子社区**：鼓励小众社区创建数据集并优化提示可以促使更精准的模型，这些社区因具备具体经验和详细反馈而在广泛数据集中常被忽视。
