### 🌟 代码贡献指南

我们欢迎提交拉取请求。在提交之前，请确保：

1. [在这里](https://github.com/open-webui/open-webui/discussions/new/choose)就您的想法展开讨论。
2. 遵循项目的编码标准，并为新功能加入测试。
3. 根据需要更新文档。
4. 撰写清晰且描述性的提交信息。

### 🛠 代码提交最佳实践：

1. **原子性提交：** 确保您的拉取请求小而专注，并且处理单一目标或任务。这有助于更轻松地进行代码审核，同时减少引入无关问题的概率。如果更改范围太大，请考虑拆分为逻辑独立的较小拉取请求。
2. **遵循现有代码约定：** 确保您的代码与项目的现有编码标准和实践一致。
3. **避免增加额外外部依赖：** 在未经事先讨论的情况下，不要包含其他外部依赖。
4. **保持框架无关性：** 我们希望保持框架无关性。尽可能自行实现功能，而不是依赖外部框架或库。如果您对这种方法有疑问或建议，请随时讨论。

### 🌐 翻译与国际化

帮助我们让 Open WebUI 面向更广泛的用户。在本节中，我们将指导您如何为项目新增翻译。

我们使用 JSON 文件来存储翻译内容。您可以在 `src/lib/i18n/locales` 目录中找到现有的翻译文件。每个目录对应特定语言，例如英文 (美国) 用 `en-US`，法语 (法国) 用 `fr-FR`，等等。您可以参考 [ISO 639 语言代码](http://www.lingoes.net/en/translator/langcode.htm) 找到特定语言的代码。

要添加一种新语言：

- 在 `src/lib/i18n/locales` 路径下创建一个新目录，其名称为适当的语言代码。例如，如果您要新增西班牙语 (西班牙) 的翻译，则创建名为 `es-ES` 的目录。
- 将美国英语的翻译文件（位于 `src/lib/i18n/locale` 中的 `en-US` 目录）复制到这个新目录，并根据您的语言更新 JSON 格式的字符串值。确保保持 JSON 对象的结构不变。
- 将语言代码及其对应的标题添加到 `src/lib/i18n/locales/languages.json` 的语言文件中。

### 🤔 问题与反馈

### 🚨 报告问题

发现问题？有想法？查看我们的 [问题标签](https://github.com/open-webui/open-webui/issues)，看看是否已经被报告或建议。如果没有，请随时创建一个新问题。当报告问题时，请遵循我们的问题模板。这些模板旨在确保从一开始就提供所有必要的详细信息，使我们能够更高效地解决您的问题。

:::important

- **模板合规性：** 请注意，如果未能遵循所提供的问题模板，或未提供请求的信息，可能会导致您的问题被关闭而不予进一步考虑。这种方法对于维护问题追踪的可管理性和完整性至关重要。

- **细节至重：** 为确保您的问题能够被理解并得到有效处理，必须包括全面的详细信息。描述应清晰，其中包括重现步骤、预期结果以及实际结果。缺乏足够的细节可能会阻碍我们解决您的问题的能力。

:::

### 🧭 支持范围

我们注意到提交的问题越来越多地与 Open WebUI 本身无关，而是与其运行环境（尤其是 Docker 设置）有关。虽然我们努力支持 Docker 部署，但了解 Docker 基础知识对获得顺畅体验至关重要。

- **Docker 部署支持：** Open WebUI 支持 Docker 部署。我们假定用户熟悉 Docker。如需了解 Docker 基础知识，请参考 [官方 Docker 文档](https://docs.docker.com/get-started/overview/)。

- **高级配置**：设置反向代理以实现HTTPS以及管理Docker部署需要具备基础知识。互联网上有众多资源可以学习这些技能。掌握这些知识将极大地提升您使用Open WebUI及类似项目的体验。
