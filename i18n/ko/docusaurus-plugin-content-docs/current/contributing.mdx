🚀 **환영합니다, 기여자님!** 🚀

Open WebUI에 기여하려는 귀하의 관심에 깊이 감사드립니다. 이 문서는 기여 절차를 안내하며, 귀하의 기여가 프로젝트를 효과적으로 개선하도록 돕습니다. 함께 Open WebUI를 더욱 멋지게 만들어봅시다!

## 💡 기여하기

기여할 의향이 있으신가요? 좋은 생각입니다! 다음 방법으로 도움을 줄 수 있습니다:

### 🌟 코드 기여 가이드라인

저희는 풀 요청을 환영합니다. 제출 전에 다음을 준수해주세요:

1. [여기](https://github.com/open-webui/open-webui/discussions/new/choose)에서 아이디어에 대한 논의를 시작하세요.
2. 프로젝트의 코딩 표준을 준수하고 새로운 기능에 대한 테스트를 포함하세요.
3. 필요에 따라 문서를 업데이트하세요.
4. 명확하고 설명적인 커밋 메시지를 작성하세요.

### 🛠 코드 PR 베스트 프랙티스:

1. **원자적 PR:** PR은 작고 집중적이며 하나의 목표 또는 작업에만 집중되어야 합니다. 이는 코드 리뷰를 쉽게 하고 관련되지 않은 문제를 도입할 가능성을 줄여줍니다. 변경 범위가 너무 커지면 논리적으로 독립적인 작은 PR로 분리하는 것을 고려하세요.
2. **기존 코드 규칙 준수:** 코드가 프로젝트의 기존 코딩 표준 및 관행과 일치하도록 하세요.
3. **추가 외부 종속성 피하기:** 사전 논의 없이 추가 외부 종속성을 포함하지 마세요.
4. **프레임워크 비종속적 접근:** 저희는 프레임워크에 종속되지 않도록 노력하고 있습니다. 가능한 한 외부 프레임워크나 라이브러리를 사용하기보다는 스스로 기능을 구현하세요. 이 접근 방식에 대해 의문이나 제안이 있다면 논의해 주세요.

기여해 주셔서 감사합니다! 🚀

### 📚 문서화 및 튜토리얼

Open WebUI를 더 쉽게 접근할 수 있도록 문서를 개선하거나, 튜토리얼 작성, 설정 및 웹 UI 최적화 안내를 작성하는 데 도움을 주세요.

### 🌐 번역 및 국제화

Open WebUI를 더 넓은 청중에게 공개하는 것을 돕는 데 기여하세요. 이 섹션에서는 프로젝트에 새로운 번역을 추가하는 과정을 안내합니다.

저희는 JSON 파일을 사용하여 번역을 저장합니다. 기존 번역 파일은 `src/lib/i18n/locales` 디렉토리에서 찾을 수 있습니다. 각 디렉토리는 특정 언어를 나타내며, 예를 들어 `en-US`는 영어(미국), `fr-FR`는 프랑스어(프랑스) 등입니다. 특정 언어에 대한 적절한 코드를 찾으려면 [ISO 639 언어 코드](http://www.lingoes.net/en/translator/langcode.htm)를 참조하세요.

새로운 언어를 추가하려면:

- 적절한 언어 코드를 이름으로 사용하여 `src/lib/i18n/locales` 경로에 새 디렉토리를 만드세요. 예를 들어, 스페인어(스페인) 번역을 추가하려면 `es-ES`라는 새 디렉토리를 만드세요.
- 미국 영어 번역 파일(들)을(`src/lib/i18n/locale`의 `en-US` 디렉토리에서) 새 디렉토리에 복사하고 JSON 형식으로 언어에 맞게 문자열 값을 업데이트하세요. JSON 객체의 구조를 반드시 유지하세요.
- 언어 파일 `src/lib/i18n/locales/languages.json`에 언어 코드와 해당 타이틀을 추가하세요.

### 🚨 문제 보고

무언가 이상을 발견했나요? 아이디어가 있나요? [이슈 탭](https://github.com/open-webui/open-webui/issues)을 확인하여 이미 보고되거나 제안되었는지 확인하세요. 그렇지 않다면 새로운 이슈를 열어주세요. 문제를 보고할 때, 제공된 이슈 템플릿을 따라주세요. 템플릿은 처음부터 필요한 세부 정보를 모두 제공하도록 되어 있어, 우려 사항을 보다 효율적으로 해결할 수 있도록 합니다.

:::important

- **템플릿 준수:** 제공된 이슈 템플릿을 따르지 않거나 요청된 정보를 제공하지 않는 경우, 추가 고려 없이 이슈가 종료될 가능성이 높습니다. 이는 이슈 추적의 관리 가능성 및 무결성을 유지하는 데 중요한 접근 방식입니다.

- **세부 정보의 중요성:** 문제를 이해하고 효과적으로 해결할 수 있도록 상세한 정보를 포함하는 것이 중요합니다. 설명은 명확해야 하며, 재현 단계, 예상 결과, 실제 결과를 포함해야 합니다. 충분한 세부 정보가 부족하면 문제를 해결하기 어려울 수 있습니다.

:::

### 🧭 지원 범위

Open WebUI와 직접적으로 관련되지 않고 주로 실행 환경(특히 Docker 설정)과 관련된 문제들이 증가하고 있음을 알게 되었습니다. Docker 배포를 지원하기 위해 노력하지만, Docker 기본 사항을 이해하는 것이 원활한 환경을 위해 중요합니다.

- **Docker 배포 지원**: Open WebUI는 Docker 배포를 지원합니다. Docker 기본 사항에 대한 지식을 전제로 합니다. Docker 기본 사항은 [Docker 공식 문서](https://docs.docker.com/get-started/overview/)를 참조하세요.

- **고급 설정**: HTTPS를 위한 리버스 프록시 설정 및 Docker 배포 관리는 기본적인 지식이 필요합니다. 이러한 기술을 배우기 위한 온라인 자원이 많이 제공되고 있습니다. 이 지식을 습득하면 Open WebUI 및 유사 프로젝트에서의 경험을 크게 향상시킬 수 있습니다.
