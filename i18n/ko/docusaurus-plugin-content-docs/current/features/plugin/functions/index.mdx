## 🚀 기능이란 무엇인가요?

기능은 Open WebUI의 **플러그인**과 같습니다. 새 AI 모델 제공자(예: Anthropic 또는 Vertex AI)를 추가하거나, 메시지를 처리하는 방법을 조정하거나, 더 나은 사용성을 위해 인터페이스에 사용자 정의 버튼을 추가하는 등 Open WebUI의 **기능을 확장**하는 데 도움을 줍니다.

복잡한 통합이 필요한 외부 도구와 달리, **기능은 내장되어 있으며 Open WebUI 환경 내에서 작동합니다.** 이는 기능이 빠르고 모듈식이며 외부 의존성에 의존하지 않음을 의미합니다.

기능은 WebUI의 작동 방식을 정확히 원하는 대로 개선할 수 있도록 하는 **모듈식 구성 요소**로 생각할 수 있습니다. 가볍고, 높은 수준의 사용자 정의가 가능하며, **순수 Python**으로 작성되어 새로운 AI 지원 워크플로우나 Google 검색 또는 Home Assistant 같은 애플리케이션과의 통합까지 원하는 것을 자유롭게 창작할 수 있습니다.

---

## 🏗️ 기능의 유형

Open WebUI에는 각기 다른 목적을 가진 **세 가지 유형의 기능**이 있습니다. 이를 하나씩 살펴보고 무엇을 하는지 자세히 설명해드리겠습니다:

---

### 1. [**파이프 기능** – 맞춤형 "에이전트/모델" 생성하기](./pipe.mdx)

**파이프 기능**은 **맞춤형 에이전트/모델** 또는 통합을 생성하는 방법으로, 인터페이스에서 마치 독립형 모델처럼 나타납니다.

**무엇을 하나요?**
- 파이프를 사용하면 복잡한 워크플로를 정의할 수 있습니다. 예를 들어, **모델 A**와 **모델 B**로 데이터를 보내고, 출력물을 처리하여 이를 하나의 최종 답변으로 결합하는 파이프를 생성할 수 있습니다.
- 파이프는 AI를 사용하지 않아도 됩니다! 검색 API, 날씨 데이터, 또는 **Home Assistant** 같은 시스템 설정도 가능합니다. 기본적으로 상호작용하고 싶은 모든 것이 Open WebUI의 일부가 될 수 있습니다.

**사용 사례 예시:**
Open WebUI에서 Google 검색을 직접 쿼리하고 싶다고 가정해보세요. 다음과 같은 파이프 기능을 생성할 수 있습니다:
1. 메시지를 검색어로 사용합니다.
2. Google 검색 API에 쿼리를 전송합니다.
3. 응답을 처리하고 WebUI 내부에서 일반적인 "모델" 응답처럼 반환합니다.

활성화하면 **파이프 기능이 자체적으로 선택 가능한 모델로 표시됩니다.** 인터페이스에서 모델처럼 작동하는 맞춤형 기능이 필요할 때는 언제든지 파이프를 사용하세요.

자세한 가이드는 [**파이프 기능**](./pipe.mdx)을 참조하세요.

---

### 2. [**필터 기능** – 입력 및 출력 수정](./filter.mdx)

**필터 기능**은 AI에 **전송되는 데이터** 또는 **AI에서 반환된 데이터**를 조정하는 도구와 같습니다.

**무엇을 하나요?**
필터는 워크플로에서 "후크" 역할을 하며 두 가지 주요 부분으로 구성됩니다:
- **인렛(Inlet)**: 모델에 보내는 입력을 조정합니다. 추가 지침, 키워드 또는 형식 조정을 추가할 수 있습니다.
- **아웃렛(Outlet)**: 모델에서 반환된 출력을 수정합니다. 응답 정리, 톤 조정 또는 데이터를 특정 스타일로 형식화할 수 있습니다.

**사용 사례 예시:**
정확한 형식이 필요한 프로젝트를 진행 중이라고 가정해봅시다. 다음을 보장하는 필터를 사용할 수 있습니다:
1. 입력이 항상 원하는 형식으로 변환됩니다.
2. 모델의 출력이 정리되어 표시됩니다.

필터는 **특정 모델에 연결되거나** 필요에 따라 **전역적으로 모든 모델에 활성화될 수 있습니다.**

자세한 예시와 설정 방법을 보려면 [**필터 기능**](./filter.mdx)을 확인하세요.

---

### 3. [**액션 기능** – 사용자 정의 버튼 추가](./action.mdx)

**액션 기능**은 채팅 인터페이스에 **사용자 정의 버튼**을 추가하는 데 사용됩니다.

**무엇을 하나요?**
액션은 채팅에서 특정 기능을 직접 트리거하는 **인터랙티브 쇼트컷**을 정의할 수 있도록 합니다. 이러한 버튼은 각 채팅 메시지 아래에 나타나며 정의한 기능을 편리하게 한 번 클릭으로 접근할 수 있습니다.

**사용 사례 예시:**
긴 메시지를 요약하거나 번역과 같은 특정 출력을 자주 생성해야 한다고 가정해봅시다. 다음 작업을 수행하는 액션 기능을 만들 수 있습니다:
1. 모든 수신 메시지 아래에 “요약하기” 버튼을 추가합니다.
2. 클릭하면 메시지를 처리하여 요약을 반환하는 사용자 정의 기능을 트리거합니다.

버튼은 정의한 확장 기능과 상호작용할 수 있는 **깔끔하고 사용자 친화적인 방법**을 제공합니다.

설정 방법은 [**액션 기능 가이드**](./action.mdx)를 참조하세요.

## 🛠️ 기능 사용 방법

Open WebUI에서 기능을 활용하는 방법은 다음과 같습니다:

### 1. **기능 설치하기**
Open WebUI 인터페이스를 통해 또는 수동으로 기능을 설치할 수 있습니다. 커뮤니티에서 생성된 기능은 [Open WebUI 커뮤니티 사이트](https://openwebui.com/functions)에서 찾을 수 있습니다.

⚠️ **주의하세요.** 신뢰할 수 있는 소스에서만 기능을 설치하세요. 알 수 없는 코드를 실행하는 것은 보안 위험을 초래할 수 있습니다.

---

### 2. **기능 활성화하기**
설치 후 기능은 명시적으로 활성화해야 합니다: 
- **파이프 기능**을 활성화하면 인터페이스에서 자체 **모델**로 사용할 수 있게 됩니다.
- **필터** 및 **액션 기능**의 경우, 활성화만으로는 충분하지 않습니다. 특정 모델에 할당하거나 모든 모델에 대해 전역적으로 활성화해야 합니다.

---

### 3. **모델에 필터 또는 액션 할당하기**
- `Workspace => Models`로 이동하여 필터나 액션을 해당 모델에 할당하세요.
- 또는, `Workspace => Functions`로 이동하여 "..." 메뉴를 선택한 다음 **Global** 스위치를 전환하여 **모든 모델에 대해 전역적으로** 기능을 활성화하세요.

---

### 빠른 요약
- **파이프**는 상호작용할 수 있는 독립적인 모델로 표시됩니다.
- **필터**는 입력/출력을 수정하여 AI 상호작용을 원활하게 만듭니다.
- **액션**은 개별 채팅 메시지에 클릭 가능한 버튼을 추가합니다.

설정 과정을 완료하면 기능이 워크플로우를 원활하게 개선할 것입니다.

---

## ✅ 기능을 사용하는 이유?

기능은 Open WebUI를 사용해 **새로운 가능성**을 열고자 하는 사람들을 위해 설계되었습니다.

- **확장**: 새로운 모델을 추가하거나 API, 데이터베이스, 스마트 디바이스와 같은 비AI 도구를 통합합니다.
- **최적화**: 입력 및 출력을 조정하여 사용 사례에 완벽히 맞출 수 있습니다.
- **단순화**: 버튼 또는 바로가기를 추가하여 인터페이스를 직관적이고 효율적으로 만듭니다.

특정 프로젝트를 위한 워크플로우를 커스터마이징하거나 외부 데이터를 통합하거나, Open WebUI를 더 쉽게 사용할 수 있도록 하는 경우, 기능은 인스턴스를 제어하는 데 핵심적인 역할을 합니다.
