# 🛠️ 도구 및 기능

Open WebUI를 처음 접했거나 이미 사용 중이지만, "도구", "기능", "파이프라인"에 대한 모든 이야기에 혼란스러울 수도 있습니다. 모든 것이 신비한 기술 용어처럼 들리죠? 걱정하지 마세요! 하나씩 분명하게, 단계적으로 살펴보겠습니다. 이 글을 끝내고 나면, 이러한 용어들의 의미, 작동 방식, 그리고 생각보다 덜 복잡하다는 사실에 대한 확실한 이해를 갖게 될 겁니다.

## 요약

- **도구**는 LLM의 능력을 확장하여 날씨, 주식 가격 등과 같은 실시간 데이터를 수집할 수 있도록 합니다.
- **기능**은 Open WebUI 자체의 기능을 확장하여 새로운 AI 모델 지원(예: Anthropic 또는 Vertex AI) 또는 가용성 개선(예: 사용자 정의 버튼 또는 필터 생성)을 가능하게 합니다.
- **파이프라인**은 Open WebUI 기능을 API 호환 워크플로로 변환하려는 고급 사용자에게 적합합니다. 주로 무거운 처리를 분산하기 위해 사용됩니다.

도구와 기능의 사용을 시작하는 것은 쉽습니다. 모든 것이 이미 코어 시스템에 내장되어 있으므로, 버튼을 클릭하고 커뮤니티에서 직접 이러한 기능을 가져오기만 하면 됩니다. 코딩이나 깊은 기술적 작업이 필요하지 않습니다.

## "도구"와 "기능"이란?

**Open WebUI**를 많은 작업을 수행할 수 있는 "기본" 소프트웨어로 생각해보세요. 그러나 때로는 기본적으로 제공되지 않는 추가 기능이나 능력이 필요합니다. 이때 **도구**와 **기능**이 등장합니다.

### 도구

**도구**는 LLM이 단순히 텍스트 처리 이상을 수행할 수 있도록 하는 흥미로운 기능입니다. 이들은 LLM 자체적으로는 불가능한 **외부 능력**을 제공합니다.

#### 도구의 예:

LLM과 대화를 나누고 실시간으로 최신 날씨 업데이트 또는 주식 가격을 알고 싶다고 상상해보세요. 일반적으로 LLM은 사전 학습된 지식을 사용하기 때문에 이렇게 할 수 없습니다. 이때 **도구**가 필요한 것이죠!

- **도구는 플러그인과 같아서** LLM이 **실시간**, **실제 데이터**를 수집할 수 있게 해줍니다. 따라서 '날씨 도구'가 활성화되면 모델이 인터넷에서 실시간 날씨 데이터를 가져와 대화 중에 표시할 수 있습니다.

도구는 본질적으로 **능력**을 AI에게 제공해 외부 세계와 상호 작용할 수 있도록 돕는 것입니다. 이를 추가하면 LLM은 대화의 맥락에 따라 유용한 정보를 "끌어오거나" 특정 작업을 수행할 수 있습니다.

#### 도구의 예 (LLM의 능력 확장):
1. **실시간 날씨 예측** 🛰️.
2. **주식 가격 조회** 📈.
3. **항공편 추적 정보** ✈️.

### 기능

**도구**가 AI가 대화 중에 사용하는 것이라면, **기능**은 Open WebUI 자체의 기능을 확장하거나 사용자 정의할 수 있도록 도와줍니다. 도구가 새로운 재료를 요리에 추가하는 것이라면, 기능은 주방을 제어하는 과정이라고 생각해보세요! 🚪

#### 이를 분해해보면:

- **기능**은 **Open WebUI** 자체에서 **기능**을 수정하거나 추가할 수 있는 능력을 제공합니다.
- LLM에게 새로운 능력을 부여하는 것이 아니라, 플랫폼 자체의 **인터페이스, 동작 또는 논리**를 확장하는 것입니다!

예를 들어 다음과 같은 작업을 원할 때:
1. WebUI에 새로운 AI 모델인 **Anthropic**을 추가.
2. 자주 사용하는 명령을 실행하는 사용자 정의 버튼을 툴바에 추가.
3. 부적절하거나 **스팸 메시지**를 텍스트에서 필터링하는 더 나은 **필터** 기능 구현.

기능이 없으면 이러한 작업은 모두 불가능할 것입니다. 그러나 Open WebUI의 이 프레임워크를 통해 이러한 기능을 쉽게 확장할 수 있습니다!

### 차이점 요약:
- **도구**는 LLM이 원래의 능력을 넘어 실시간 정보 검색이나 외부 데이터를 기반으로 한 맞춤형 작업 수행 등 **더 많은 것을 할 수 있게** 만듭니다.
- **기능**은 WebUI 자체가 **더 많은 일을 할 수 있도록** 도와줍니다(예: 새로운 AI 모델 추가 또는 데이터를 필터링하는 더 스마트한 방법 만들기).

둘 다 **플러그인 방식**으로 설계되어 있어 커뮤니티에서 클릭 한 번으로 시스템에 쉽게 가져올 수 있습니다! 🎉 코딩이나 조정에 수시간을 소비하지 않아도 됩니다.

## 파이프라인이란?

그리고 **파이프라인**이 있습니다… 여기서는 약간 기술적으로 들릴 수 있습니다. 하지만 절망하지 마세요.

**파이프라인**은 Open WebUI가 OpenAI의 API 시스템과 **상호운용 가능**하게 만드는 것을 목표로 하는 이니셔티브의 일부입니다. 본질적으로, 이들은 **도구**와 **기능**이 이미 제공하는 것을 확장하지만, 훨씬 더 유연하게 만듭니다. 기능을 OpenAI API 호환 형식으로 변환할 수 있게 합니다. 🧠

### 하지만 중요한 점은...

초고급 설정을 다루지 않는 한, 파이프라인이 **필요하지 않을** 가능성이 높습니다.

- **파이프라인은 누구를 위한 것인가?** 일반적으로 **전문가** 또는 더 복잡한 사용 사례를 운영하는 사람들을 위한 것입니다.
- **언제 필요할까요?** 기본 Open WebUI 인스턴스에서 처리 작업을 다른 기기로 오프로드하려고 할 때 (즉, 기본 시스템에 과도한 부하를 주지 않기 위함).
  
대부분의 경우, 초보자 또는 중급 사용자로서는 파이프라인에 대해 걱정할 필요가 없습니다. **도구**와 **기능**이 Open WebUI 경험에 제공하는 혜택을 즐기는 데만 집중하세요!

## 시도해보고 싶나요? 🚀

Open WebUI로 들어가서 커뮤니티 섹션으로 이동하여 **날씨 업데이트** 같은 도구를 가져오거나, 툴바에 새로운 기능을 추가해보세요. 이러한 도구를 탐색하면 Open WebUI가 얼마나 강력하고 유연한지 알게 될 것입니다!

🌟 배울 것이 항상 더 많으니, 호기심을 유지하며 계속 실험해보세요!
