Open WebUIでは、ユーザーに最高の体験を提供するために、プラットフォームの改善を継続的に行っています。以下は、現在進行中および将来の開発に関する構造化されたロードマップで、インターフェース、情報検索、およびコミュニティに分類されています。

## インターフェース 🖥️

インターフェースの革新に関するロードマップは、高度に直感的でアクセス可能なプラットフォームを作ることを目的としています。これにより、ユーザー満足度が向上するだけでなく、ユーザーの認知的負担を軽減することで生産性も向上します。

- 📦 **単一バイナリーの実行可能ファイルのパッケージ化**: 配置を簡素化し、異なるプラットフォーム間での互換性を確保するために、インターフェースを単一のバイナリー実行可能ファイルとして提供することを目指しています。このアプローチにより、設置と更新が容易となり、特に技術的専門知識が限られたユーザーにとって使いやすさが大幅に向上します。

- 👤 **ユーザーページ**: ユーザーが投稿を作成できる個人のユーザーページ機能。フォロワー、いいね、コメントなどの機能も含まれます。これにより、ユーザーはモデルの設定、プロンプト、ファイルを幅広いコミュニティと効果的に共有でき、プラットフォーム周辺に豊かでつながりのあるエコシステムを作り出します。

- 📝 **AIによるノート作成**: NotionやObsidianのようなツールに触発され、AI統合を含む強力なノート作成機能を導入する予定です。シンプルなノート作成から本格的な文書作成まで、プラットフォーム内でシームレスな体験を提供します。

- 📈 **高度なユーザートラッキングとコスト管理ツール**: アプリケーションのパフォーマンスとユーザーアクティビティを追跡し、効果的にコストを管理するための包括的なツールにアクセスできるようになります。これらのツールは、ユーザーに情報を提供して意思決定を助け、経験を改善し、予算管理を可能にし、AIアプリケーションのリソース使用を最適化します。

- 🧠 **AIワークフローツール**: ノードベースのツールを使用してAIシステムの複数の側面を調整・構成します。このツールにより、異なるAIモジュールやサービスを視覚的に接続し、複雑なワークフローを簡単に作成できます。深いAIプログラミング知識を必要とせずにAIを最大限に活用できるよう設計されています。

- 🔊 **ローカルテキスト読み上げ統合**: この機能により、自然言語処理を使用してテキストをリアルな音声オーディオに変換し、視覚障がいのあるユーザーに特に優れたアクセス性を提供します。画面操作なしでも情報を消費できるため、マルチタスクが可能になりユーザーのエンゲージメントが向上します。

- 📣 **ウェイクワード検出**: 音声コマンドによるハンズフリー操作の導入により、アクセス性が向上するだけでなく、IoTやスマート環境の新興トレンドにも対応し、プラットフォームを将来に向けて準備します。

- 💻 **より良いコード実行**: プラットフォーム内でのコーディング機能を強化することで、幅広い開発活動をサポートし、開発者にとってのワンストップソリューションとなり、より幅広い技術系ユーザーを引き付け、強固な開発者コミュニティを育成します。

- 🤖 **コードインタープリタ機能**: プラットフォームに埋め込まれた柔軟なコードインタープリタは、複数のプログラミング言語をサポート可能で、外部ツールへの依存を減らし、開発プロセスを簡素化して効率を向上させます。

- 🔧 **洗練されたファインチューンサポート**: Open WebUIでは、ユーザーがインターフェースを使用して応答を評価するだけで、自動的にファインチューンデータセットを構築できるシームレスなファインチューニングプロセスを提供します。データセットを準備するためのPythonノートブックも提供し、そのノートブックから直接ファインチューンできる状態にします。このアプローチは非常にシンプルで、エレガントにユーザーのワークフローへファインチューニングを統合します。


## 情報検索（RAG） 📚

Open WebUIは現在、既存の情報検索（IR）モデルを効率的に使用できるインターフェースを提供しています。しかし、改善の余地は大いにあると考えています。WebUIにおける現在のRAG（検索強化生成）実装の限界を十分に認識しており、画一的なアプローチではすべてのユーザーのニーズを効果的に満たすことはできません。

これに対応するため、フレームワークを高度にモジュール化し、大幅に設定可能に変革することを約束します。新しい方向性により、ユーザーは情報検索プロセスを簡単に特定の要件に合わせてカスタマイズできるようになり、シンプルでドラッグ＆ドロップが可能なインタラクティブなインターフェースを使用できます。この改良により、より柔軟で個別化された情報検索セットアップを提供し、ユーザー体験を大幅に向上させることを目指しています。

こちらは情報検索能力を向上させるための焦点を絞った計画です：

- 🌐 **カスタマイズ可能なRAGフレームワーク**: ユーザーフレンドリーなインターフェイスを通じて、情報検索の設定を手軽にカスタマイズできる未来を描いています。これには、ユーザーが深い技術的知識を必要とせずにドラッグ、ドロップ、設定できるモジュールコンポーネントが含まれます。

- 🔄 **SoTA（最新技術）手法との高度な統合**: 最新の検索手法の進歩を統合し、高度な技術とアーキテクチャを使用して正確性と効率性を向上させます。

- 🔍 **専用の研究開発**: 現在の方法の限界を押し広げる革新的な検索方法を発見・統合するために、主要な研究機関と協力しながら研究開発を拡大します。

## コミュニティ 🤝

知識と進展を共有し、すべての人が貢献し利益を得ることができる活気あるコミュニティを作りたいと考えています：

- 🏆 **LLMリーダーボード**: モデルの評価とランキングを透明性を維持しつつリアルワールドの応用を活用して性能をベンチマークするコミュニティ主導のアプローチ。これにより、実際の実用性を反映しない標準的なデータセットへの依存が排除されます。

- 👥 **サブコミュニティ**: ニッチなコミュニティを奨励してデータセットを収集しプロンプトを最適化することで、これらのコミュニティが持つ特定の洞察や詳細なフィードバックを活かし、より調整されたモデルを生み出せる可能性があります。

🔗 **コミュニティホワイトペーパー**: 詳細な戦略を探り、AI駆動型プラットフォームの未来を共に形作るために[ホワイトペーパー](https://openwebui.com/assets/files/whitepaper.pdf)をご覧ください。
